# Blinkit_Dashboard

1. project objective(business problem)


The objective of this project is to analyze Blinkit’s sales performance and identify key factors affecting revenue, customer satisfaction, and product distribution.
The goal is to help the business understand which products, outlet types, and locations perform best so they can make better decisions on inventory and expansion.

2. kpi & metrics(KPI indicator is a measurable value that shows how well a business or process is performing)

The following key performance indicators (KPIs) were created:
Total Sales → overall revenue
Average Sales → average revenue per item
Number of Items → total inventory
Average Rating → customer satisfaction
These KPIs help stakeholders quickly understand the overall business health.”

3.data sources and transformation


I imported the dataset from an Excel file into Microsoft Power BI. Then I used Power Query Editor to clean and transform the data by handling missing values,
correcting data types, removing duplicates, and standardizing columns. This ensured the dataset was accurate and ready for analysis.”

4. dax and advanced features

I used DAX to create calculated measures such as:
Total Sales = SUM of sales
Average Sales = AVERAGE
Average Rating
I also implemented:
Dynamic measures
Aggregations
Field parameters (to switch metrics dynamically)
These enhanced interactivity and flexibility of the dashboard.”

5. visualization and story telling

I designed the dashboard in Microsoft Power BI with a focus on clarity and usability:
KPI cards for quick insights
Bar charts for item type analysis
Donut charts for outlet size distribution
Line chart for trend analysis
Table for detailed outlet performance
I also added slicers for Outlet Size, Location, and Item Type to allow users to explore data interactively.

6. Business impact

This dashboard helps the business understand which products and outlets are performing well. It shows sales, customer ratings, and item distribution in a clear way.

Using this, the company can stock the right products, focus on high-performing locations, and improve customer satisfaction. Overall, it helps in making better business decisions based on data


7. Why I used these graphs?


I selected each graph based on the type of data and the insight I wanted to show. Different visuals are better suited for comparison, distribution, and trends, so I used them accordingly in Microsoft Power BI.”
KPI Cards
“I used KPI cards to show key metrics like total sales and average rating because they give a quick summary of overall performance.”
Bar Chart (Item Type)
“I used a bar chart to compare sales across different item categories, as it is best for comparing multiple categories clearly.”
Donut Chart (Outlet Size)
“I used a donut chart to show the proportion of sales by outlet size, which makes it easy to understand distribution.”
Line Chart (Trend)
“I used a line chart to show sales trends over time, which helps identify growth patterns.”
Table
“I used a table to provide detailed information that cannot be easily shown in charts.”
Slicers
“I used slicers to allow users to filter data interactively and analyze specific segments.”
